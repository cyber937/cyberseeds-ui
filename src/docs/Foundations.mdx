# Cyberseeds UI — Design Foundations

Cyberseeds UI is built on a consistent set of design tokens that ensure visual harmony across all components.

## Color System (v1.0.0)

All colors are applied via **CSS custom properties**, enabling both preset and custom colors to follow the same code path.

Semantic mappings can be customized via `UIColorProvider`:

```tsx
<UIColorProvider
  initialColor="success"
  semanticColors={{ success: "emerald", error: "rose" }}
>
```

### Custom Colors

Provide an object with OKLCH or hex values:

```tsx
const brand: CustomColor = {
  base: "#4f46e5",     // Primary shade
  hover: "#4338ca",    // Hover state
  active: "#3730a3",   // Active/pressed
  focus: "#4f46e5",    // Focus ring
  light: "#e0e7ff",    // Light background (PillBox, etc.)
  lightText: "#312e81", // Text on light background
  border: "#a5b4fc",   // Border color
};

// Or provide only `base` for automatic shade generation:
const simple: CustomColor = { base: "#4f46e5" };
```

## Spacing & Layout

- **Tailwind CSS v4** with `cs:` namespace prefix
- Touch targets: minimum 44×44px on mobile (`cs:max-md:min-h-11`)
- Responsive breakpoints: `sm` (640px), `md` (768px), `lg` (1024px), `xl` (1280px)

---

## Typography

- Font family: `cs:font-sans` (system UI stack)
- Font weights: `cs:font-medium` (buttons, labels), `cs:font-normal` (body)

---

## Motion

- Fast transitions: `150ms ease-in-out` (hover, focus)
- Normal transitions: `200ms ease-in-out` (color changes)
- Slow transitions: `300ms ease-in-out` (layout shifts)
- Respects `prefers-reduced-motion` via `cs:motion-reduce:transition-none`

---

## Theming

### Dark Mode

Use `ThemeProvider` to manage theme:

```tsx
<ThemeProvider mode="system">
  <App />
</ThemeProvider>
```

Modes: `"light"` | `"dark"` | `"system"`

### UIColorProvider

Sets the accent color for all child components:

```tsx
<UIColorProvider initialColor="indigo">
  <Button>Uses indigo</Button>
  <Input label="Also indigo" />
</UIColorProvider>
```
